test
语法 test(选项)

选项 
-b<文件>：如果文件为一个块特殊文件，则为真； 
-c<文件>：如果文件为一个字符特殊文件，则为真； 
-d<文件>：如果文件为一个目录，则为真； 
-e<文件>：如果文件存在，则为真； 
-f<文件>：如果文件为一个普通文件，则为真；
-r<文件>：如果文件可读，则为真；
-w<文件>：如果文件可写，则为真； 
-x<文件>：如果文件可执行，则为真。
-s<文件>：如果文件的长度不为零，则为真；

-g<文件>：如果设置了文件的SGID位，则为真； 
-G<文件>：如果文件存在且归该组所有，则为真； 
-k<文件>：如果设置了文件的粘着位，则为真； 
-O<文件>：如果文件存在并且归该用户所有，则为真； 
-p<文件>：如果文件为一个命名管道，则为真； 
-S<文件>：如果文件为一个套接字特殊文件，则为真； 
-u<文件>：如果设置了文件的SUID位，则为真；

实例 
linux中shell编程中的test常见用法： 
判断表达式 
if test                  #表达式为真 
if test !                #表达式为假 
test 表达式1 –a 表达式2 #两个表达式都为真时，则为真； 
test 表达式1 –o 表达式2 #两个表达式有一个为真时，则为真；
test 表达式1 !  表达式2  #条件求反

判断字符串 
test –n 字符串 #字符串的长度为非零时，则为真； 
test –z 字符串 #字符串的长度为零时，则为真； 
test 字符串1  ＝ 字符串2 #两个字符串相同时，则为真；
test 字符串1 !＝ 字符串2 #两个字符串不同时，则为真；

判断整数 
test 整数1 -eq 整数2 #两个整数相等时，则为真；
test 整数1 -ne 整数2 #两个整数不相等时，则为真；
test 整数1 -gt 整数2 #整数1大于整数2时，则为真；
test 整数1 -ge 整数2 #整数1大于等于整数2时，则为真； 
test 整数1 -lt 整数2 #整数1小于整数2时，则为真；
test 整数1 -le 整数2 #整数1小于等于整数2时，则为真；

判断文件 
test File1 –ef File2 #两个文件为同一个文件时，则为真，可用于硬连接。
                      #主要判断两个文件是否指向同一个inode。 
test File1 –nt File2 #文件1比文件2新时，则为真；（时间的比较） 
test File1 –ot File2 #文件1比文件2旧时，则为真；（时间的比较）
test –b File #文件存在并且是块设备文件时，则为真；
test –c File #文件存在并且是字符设备文件时，则为真；
test –d File #文件存在并且是目录时，则为真； 
test –e File #文件存在时，则为真； 
test –f File #文件存在并且是正规文件时，则为真；
test –r File #文件存在并且可读时，则为真；
test –w File #文件存在并且可写时，则为真；
test –x File #文件存在并且可执行时，则为真；
test –s File #文件存在并且是非空白文件时，则为真；
test –h File #文件存在并且是一个符号链接（同-L）时，则为真；
test –L File #文件存在并且是一个符号链接（同-h）时，则为真；

test –g File #文件存在并且设置了组id时，则为真；
test –G File #文件存在并且属于有效组ID时，则为真；
test –k File #文件存在并且设置了Sticky bit位时，则为真；
test –o File #文件存在并且属于有效用户ID时，则为真； 
test –p File #文件存在并且是一个命名管道时，则为真；
test –t FD   #文件描述符是在一个终端打开时，则为真；
test –u File #文件存在并且设置了它的set-user-id位时，则为真；
